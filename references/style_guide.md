# Style Guide

Use this reference when writing or reviewing text for this test skill.

## Rules

- Keep `SKILL.md` short and action-oriented.
- Put optional detail in `references/` instead of the main skill body.
- Prefer imperative instructions over background explanation.
- Use concrete file names and commands when describing validation steps.
- Avoid copying a large production skill structure into this test fixture.

## Anti-Patterns

- Long conceptual introductions before the workflow.
- Duplicate instructions in `SKILL.md` and `README.md`.
- Hidden dependencies that are not mentioned in the validation section.
- Scripts that modify files during validation without an explicit user request.
